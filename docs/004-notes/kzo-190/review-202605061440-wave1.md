## LOW

### LOW-1: `apps/api/test/unit/catalog-sync-worker.test.ts:20-24` — `catalogProvider` mock missing `supportsMetadataEnrichment`

**What's wrong:**  
The `createDeps()` factory constructs an inline `catalogProvider` mock:

```ts
const catalogProvider = {
  reserveCapacity: vi.fn(),
  fetchInstrumentCatalog: vi.fn(),
  fetchDelistingHistory: vi.fn(),
  // ← supportsMetadataEnrichment: false  MISSING
};
```

`InstrumentCatalogProvider` now requires `readonly supportsMetadataEnrichment: boolean`. The omission is hidden from TypeScript by `catalogRegistry: deps.catalogRegistry as never` at each call site (L46, L72, L96, L139, L171). No runtime failure occurs because `createCatalogSyncHandler` never reads `supportsMetadataEnrichment`. Tests pass.

**Why it matters:**  
Per Step 7 of the scope-todo: "Add `supportsMetadataEnrichment: false|true` to any catalog-provider mock construction." Also per `.claude/rules/interface-caller-verification.md` companion note: if a future refactor makes catalog-sync-worker read the field, it silently gets `undefined` instead of `false`. The `as never` cast suppresses the type guard that would normally catch this.

**Fix:**  
Add `supportsMetadataEnrichment: false` to the `catalogProvider` inline object (this mock represents a TW provider).

```ts
const catalogProvider = {
  supportsMetadataEnrichment: false,   // KZO-190: TW catalog provider is a no-op
  reserveCapacity: vi.fn(),
  fetchInstrumentCatalog: vi.fn(),
  fetchDelistingHistory: vi.fn(),
};
```

### LOW-2: `apps/api/test/unit/catalogSyncReschedule.test.ts:33-38` — `createProvider()` factory missing `supportsMetadataEnrichment`

**What's wrong:**  
The `createProvider()` utility at L33-38 is used to create both `twProvider` and `usProvider` in `createDeps()`:

```ts
function createProvider() {
  return {
    reserveCapacity: vi.fn(),
    fetchInstrumentCatalog: vi.fn(),
    fetchDelistingHistory: vi.fn(),
    // ← supportsMetadataEnrichment: false  MISSING
  };
}
```

Same protection/risk profile as LOW-1. Both providers represent TW/US markets (→ `false`). The `catalogRegistry as never` cast at each handler construction call suppresses the TypeScript error.

**Fix:**  
Add `supportsMetadataEnrichment: false` to the returned object:

```ts
function createProvider() {
  return {
    supportsMetadataEnrichment: false,   // KZO-190: TW/US catalog providers are no-ops
    reserveCapacity: vi.fn(),
    fetchInstrumentCatalog: vi.fn(),
    fetchDelistingHistory: vi.fn(),
  };
}
```
