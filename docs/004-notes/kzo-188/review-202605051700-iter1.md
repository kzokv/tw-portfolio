## INFORMATIONAL

- **`InstrumentCombobox.tsx` ‚Äî `liveResults` Map is deliberately NOT cleared when `marketCodeFilter` changes away from `"AU"`** ‚Äî The Map (`liveResults: Map<string, TransactionInstrumentOption>`) is a session cache keyed by `${ticker}|${marketCode}`. It persists across filter changes so `selectedInstrument` can render the display label for a previously committed live pick (e.g. "CBA ‚Äî Commonwealth Bank of Australia") even after a chip change. `liveOptions` (the search results state) IS correctly reset to `[]` by the effect's early-return path. This design matches scope-todo Phase 4 intent. No action needed.

- **`InstrumentCatalogSheet.tsx:296-301` ‚Äî defensive `|| "AU"` and `|| "pending"` fallbacks** ‚Äî `marketCode: instrument.marketCode || "AU"` and `barsBackfillStatus: instrument.barsBackfillStatus || "pending"` are dead-code paths in practice because the API route always stamps both fields on search results. The guards are harmless and would produce correct AU-specific behavior even if the route ever dropped those fields. No action needed.
