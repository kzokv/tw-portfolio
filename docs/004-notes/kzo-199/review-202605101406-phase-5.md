## Bonus: URL-sync `window.history.replaceState` pattern audit

`handleTabChange` (lines 106–121) uses a dual-write strategy: `window.history.replaceState` (synchronous, immediate URL update for `page.url()` in E2E) followed by `router.replace(url, { scroll: false })` (async Next.js router state sync).

**No feedback loop risk:**
1. `handleTabChange` fires on tab click → `setActiveTab(next)` + `replaceState` + `router.replace`
2. `router.replace` eventually resolves → Next.js updates `searchParams`
3. `useEffect([searchParams, activeTab])` fires → `fromUrl === activeTab` (already set in step 1) → guard blocks → `setActiveTab` NOT called → no re-render loop

**SSR safety:** `window.history.replaceState` is guarded by `typeof window !== "undefined"` ✓

**No double-update:** `replaceState` mutates the History API entry in-place (same history slot, no back-button pollution). `router.replace` is Next.js's equivalent — both write to the same URL, not different ones. The pair is safe by design.
