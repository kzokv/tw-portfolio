## Findings

### HIGH

#### HIGH-1: Wrong `RateLimitedError` constructor in integration test T4

**File:** `apps/api/test/integration/auLicMetadataDelegation.integration.test.ts:218`

**Problem:**  
```ts
const rateLimitErr = new RateLimitedError(30_000);
```
`RateLimitedError` constructor signature is `({ msUntilAvailable }: { msUntilAvailable: number })`.
Passing a bare number destructures to `{ msUntilAvailable: undefined }`. The instance is created
with `msUntilAvailable = undefined`, so `retryAfterSeconds` returns `1` instead of `30`.

This is the **T4 test** — the critical regression guard that verifies `RateLimitedError` propagates
through the Yahoo delegate path (per `.claude/rules/typed-transient-error-catch-audit.md`). The
`instanceof` check still passes at runtime, so T4 appears green, but the error shape is wrong.
Any future test that asserts `err.msUntilAvailable` or `err.retryAfterSeconds` against this instance
would fail with no clear cause.

The file is **not** in `apps/api/test/tsconfig.json` include list, so `tsc --noEmit` silently
skips it. The unit test file (`twelveDataAuProvider.test.ts`) uses the correct form:
```ts
new RateLimitedError({ msUntilAvailable: 30_000 })
```

**Fix:**
```ts
// Line 218 — change:
const rateLimitErr = new RateLimitedError(30_000);
// to:
const rateLimitErr = new RateLimitedError({ msUntilAvailable: 30_000 });
```

### MEDIUM

#### MEDIUM-1: Missing unit test for `/etf` MIC validation failure path

**File:** `apps/api/test/unit/twelveDataAuProvider.test.ts`

**Problem:**  
Production code in `twelveDataAu.ts` validates MIC for BOTH `/stocks` AND `/etf` responses before
accepting each instrument. The unit test file covers:
- MIC mismatch for a `/stocks` row (triggers `warn()` + skip)

But there is no test case for:
- An ETF row returned by the `/etf` endpoint with a non-`XASX` MIC

The guard exists in production (confirmed in source). Its absence from unit tests means a future
regression (e.g. accidentally removing the ETF MIC check) would be invisible to the unit suite.

**Fix:**  
Add one test case to the `/etf` describe block:  
```ts
it("skips ETF rows with non-XASX mic (MIC validation on /etf path)", async () => {
  const ETFS_WITH_WRONG_MIC = [
    { symbol: "STW", mic_code: "XNAS", type: "ETF" },  // should be XASX
  ];
  // ... mock /etf to return ETFS_WITH_WRONG_MIC, assert result is []
});
```

### LOW

#### LOW-1: Three new KZO-194 test files are outside `apps/api/test/tsconfig.json` include list

**Files:**
- `apps/api/test/unit/twelveDataAuProvider.test.ts`
- `apps/api/test/integration/auCatalogSyncTwelveData.integration.test.ts`
- `apps/api/test/integration/auLicMetadataDelegation.integration.test.ts`

**Problem:**  
`apps/api/test/tsconfig.json` is scoped to specific files in its `include` array. None of the three
new KZO-194 test files are listed. Type errors in these files are silently invisible to
`npm run typecheck` — exactly the mechanism by which HIGH-1 went undetected. Per
`.claude/rules/code-review-before-pr.md` ("Before claiming typecheck green"), files outside every
tsconfig chained into `npm run typecheck` have no coverage.

**Fix:**  
Add the three files to `apps/api/test/tsconfig.json` `include`:
```json
"include": [
  ...existing entries...,
  "unit/twelveDataAuProvider.test.ts",
  "integration/auCatalogSyncTwelveData.integration.test.ts",
  "integration/auLicMetadataDelegation.integration.test.ts"
]
```

This is LOW (not MEDIUM) because vitest catches runtime errors and all three files use fairly
straightforward types — HIGH-1 is the only currently-known error. But the gap is a latent class
of problem that should be closed.

### INFORMATIONAL

#### INFO-1: Acceptable deviation — MockTwelveDataAuCatalogProvider `yahooFallback` typed as interface

**Detail:** `MockTwelveDataAuCatalogProvider` constructor accepts `yahooFallback: InstrumentCatalogProvider`
(the interface) rather than the concrete `YahooFinanceAuMarketDataProvider`. This diverges from the
production `TwelveDataAuCatalogProvider` which also takes the interface. Implementer noted this
as a deviation. It is correct — both the real and mock classes should depend on the interface, not
the concrete class. No fix required.

#### INFO-2: Acceptable deviation — registry mock-branch widened to `AU_CATALOG_PROVIDER_MOCK || !TWELVE_DATA_API_KEY`

**Detail:** When `TWELVE_DATA_API_KEY` is absent, the registry falls back to
`MockTwelveDataAuCatalogProvider`. This is consistent with the `FinMindProvider` precedent
(same fallback pattern when `FINMIND_API_TOKEN` is absent). Guards against accidental null-key
requests in dev/CI. No fix required.

#### INFO-3: Acceptable deviation — MIC validation occurs before `/etf` fetch

**Detail:** Scope-todo implied "fetch /stocks and /etf, then validate". Implementer runs MIC
validation on `/stocks` response before calling `/etf`. The `/etf` endpoint is also validated
(both paths are covered). Fail-fast behavior is arguably better. No fix required.

#### INFO-4: `AU_RESERVED_INSTRUMENTS` fully purged — CLEAN

**Detail:** `grep -r "AU_RESERVED_INSTRUMENTS"` returns zero runtime references. Only JSDoc/comment
references remain in migration-history prose. The constant and its export have been removed. No
fix required.
