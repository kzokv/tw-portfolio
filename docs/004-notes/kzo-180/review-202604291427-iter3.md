Iter 1's Validator notes the same pattern: "First full run had one monitored-tickers timeout; targeted rerun and full rerun passed."

The flake pattern: SSE heartbeat or synthetic event timing exceeds 15s under cold-cache load. CI handles it with `retries: 2`. Local development gets `retries: 0` by default; setting `CI=1` aligns local retry behavior with CI.

**Promotion path (out of scope here):** investigate whether the SSE EventSource's heartbeat interval should be tightened, or whether the test should poll with longer timeouts when running cold. Track as a follow-up E2E-infra ticket.
