## Findings Fixed

### P1 - Draft posting could partially commit across stores

`post_transaction_draft_rows` originally saved the accounting snapshot before draft row and batch version writes were guaranteed. A late optimistic conflict could leave transactions posted while draft rows remained unconfirmed.

Fix: added `confirmAiTransactionDraftPosting(...)` to the persistence contract. Postgres now commits row confirmations, batch version, accounting snapshot, and confirmation event in one transaction. The service releases idempotency only for failures before that commit succeeds.

### P2 - E2E AAA specs violated declarative assertion shape

The new AI connector and AI Inbox specs initially put direct `expect(...)` and conditional assertion logic in test bodies.

Fix: moved those checks behind helpers outside the test body and promoted the rule to `.claude/rules/e2e-aaa-guardrails.md`.

### P2 - Lint/typecheck drift during review

Review caught unused variables and an accidental aggregate removal while fixing route code. The full API unit suite also exposed two cold-import timeout sensitivities.

Fix: restored the route aggregate usage, removed unused locals, and raised the timeout for the two load-sensitive cold-import tests.

### P2 - Playwright startup and worker contention

The full gate exposed two infrastructure flakes after the code fixes: API webServer startup exceeded the prior 60-second Playwright readiness window, and both dev-bypass and OAuth suites timed out when two local workers contended for the same web/API stack.

Fix: raised Playwright webServer readiness to 120 seconds and serialized both local E2E suites to one worker. The per-test timeouts are unchanged.
