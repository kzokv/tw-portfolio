## Findings Fixed

### P1: MCP in-process tests leaked transport close incompatibility

`mcp.integration.test.ts` could throw `TypeError: socket.destroySoon is not a function` through the Hono node transport under Fastify injection. Added a compatibility shim before `transport.handleRequest()` and registered app close cleanup for active MCP transports.

Validation: targeted MCP integration passed, then full API/package and host integration gates passed.

### P1: AI draft row patch could not clear nullable fields

Frontend row edits and backend patch normalization dropped `null` values, so optional fields such as `commissionAmount`, `taxAmount`, `note`, and `sourceSnippet` could not be cleared. Updated the frontend patch contract and backend patch filter to preserve explicit `null` while still ignoring `undefined`.

Validation: added integration coverage for nullable row clears; API package suite and full host integration passed.

### P2: AI Inbox selected-batch refresh could revert the user selection

`AiInboxPanel` tied the list loader callback to `detail?.batch.id`, which could cause an initial batch selection to be re-applied after the user changed selection. Replaced this with a `currentBatchIdRef` so refreshes preserve the active user selection.

Validation: web unit, typecheck, and E2E gates passed.

### P2: Archive/delete did not emit draft update events

Batch archive/delete changed state but did not publish the same AI draft update event family used by the inbox refresh path. Added `ai_transaction_draft_updated` publication for both actions.

Validation: API and E2E gates passed.

### P3: Connector revoke and draft destructive actions needed confirmation

Added confirmation prompts before connector revoke, draft archive, and draft delete actions to avoid accidental destructive state changes.

Validation: lint, typecheck, and E2E gates passed.
