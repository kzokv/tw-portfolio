Per-provider Re-run tooltip — copy catalog

Reuses apps/web/components/ui/TooltipInfo.tsx (Radix-based, already used in TopBar + AddTransactionCard). Trigger sits next to each provider name. Hover to expand; demonstrated below as always-open for readability.

Cooldown values shown below ("60s", "30 min") are illustrative renders. Production tooltip strings carry a {cooldown} placeholder; the value is fetched per-provider from app_config via getEffectiveProviderRerunCooldownMs(providerId), returned to the client as ProviderHealthStatusDto.rerunCooldownMs, and formatted at render time by formatCooldownLabel(ms). Admins changing the override in app_config see the new value on next page load.
