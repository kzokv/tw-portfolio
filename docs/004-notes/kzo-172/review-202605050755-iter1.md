## Security Analysis (`/senior-security`)

### SSRF
No SSRF risk. `YahooFinanceAuMarketDataProvider` calls `yahoo-finance2` SDK methods (`chart`, `quote`, `search`) with programmer-controlled symbol/query strings. The SDK resolves URLs internally. No user-supplied URL is passed to `fetch()` or any HTTP client directly.

### CRLF / Injection in search `q`
Zod schema: `.string().trim().min(2).max(50).regex(/^[A-Za-z0-9 .&'()-]+$/)`. The character class blocks `\n`, `\r`, `<`, `>`, `;`, `|`, and all non-ASCII. Firehose guard (min(2)) plus max(50) bound. **Pass.**

### Rate-limit bucket isolation
`marketDataSearchBuckets` (`marketDataSearchRateLimit.ts`) is a separate `Map<string, number[]>` from `marketDataPriceBuckets`. Default limits: search 20/min, price 30/min, both configurable independently via separate env vars. `registerMarketDataSearchEviction(app)` pairs `setInterval` + `onClose` per the eviction-lifecycle rule. **Pass.**

### XSS (web layer)
No `dangerouslySetInnerHTML` added. Search results are rendered as plain React elements. `RawInstrumentInfo.name` is a trusted-provider-derived string; it is never rendered in a context that would execute scripts. **Pass.**

### Auth enforcement
Search route in `registerRoutes.ts` calls `resolveUserId(req)` before all processing — the same guard used on `/market-data/price`. Per-IP rate limit fires before the provider call. Provider budget exhaustion returns 503 + `Retry-After`. **Pass.**

### ToS startup warning
`apps/api/src/app.ts` emits `yahoo_finance_tos_notice` at `warn` level when `!Env.AU_PROVIDER_MOCK`. No `userCount` persistence call, no DB access at startup for this path. **Pass.**

### [HIGH-1] Integration test `makeHandlerDeps` missing `persistence: persistence!` — T1 metadata-enrichment assertion will fail

**File:** `apps/api/test/integration/auStockBackfill.integration.test.ts`  
**Lines:** 150–176 (the `makeHandlerDeps()` function)  
**Owner:** QA / Backend Implementer  

**Rationale:**

`BackfillWorkerDeps` (defined in `backfillWorker.ts` line 65) requires:
```ts
persistence: { upsertInstrumentCatalog(instruments, delistings): Promise<CatalogSyncResult> }
```

`makeHandlerDeps()` currently returns:
```ts
{
  pool,
  marketDataRegistry,
  catalogRegistry,
  eventBus: { publishEvent: vi.fn()... },
  boss: { send: vi.fn()... },
  updateBackfillStatus: async (ticker, status) => { await persistence!.updateBackfillStatus(...); },
  getUsersMonitoringTicker: vi.fn()...,
  log: { ... },
  // ← persistence is ABSENT
}
```

`createBackfillHandler(deps)` destructures `persistence` at line 94 of `backfillWorker.ts`. When the metadata-enrichment block executes:
```ts
await persistence.upsertInstrumentCatalog([catalogRow], []);
```
…it throws `TypeError: Cannot read properties of undefined (reading 'upsertInstrumentCatalog')`. This TypeError is caught by the surrounding warn-and-continue block:
```ts
} catch (metaErr) {
  if (metaErr instanceof RateLimitedError) throw metaErr;
  log.warn({ ... }, "backfill_metadata_fetch_failed: continuing without enrichment");
}
```

The instrument row remains at `name = "BHP placeholder"` (from `seedAuInstrument`). T1's assertion:
```ts
expect(enrichedRow.rows[0]!.name).toBe("BHP Group Limited");
```
…will fail when the suite runs against a real Postgres backend.

**Recommended fix:**

```ts
function makeHandlerDeps(provider: ...) {
  const marketDataRegistry = new Map<MarketCode, typeof provider>();
  marketDataRegistry.set("AU", provider);
  const catalogRegistry = new Map<MarketCode, typeof provider>();
  catalogRegistry.set("AU", provider);
  return {
    pool,
    marketDataRegistry,
    catalogRegistry,
    persistence: persistence!,               // ← ADD THIS LINE
    eventBus: { publishEvent: vi.fn().mockResolvedValue(undefined) },
    boss: { send: vi.fn().mockResolvedValue(undefined) },
    updateBackfillStatus: async (ticker: string, status: string) => {
      await persistence!.updateBackfillStatus(ticker, status as ...);
    },
    getUsersMonitoringTicker: vi.fn().mockResolvedValue([]),
    log: { info: vi.fn(), warn: vi.fn(), error: vi.fn() },
  };
}
```

`persistence!` is the PostgresPersistence instance constructed in `beforeEach` (line 113 of the same file) and closed in `afterEach`. It exposes `upsertInstrumentCatalog` through the `InstrumentCatalogProvider` pathway that `PostgresPersistence` implements. The `updateBackfillStatus` closure already references the same `persistence!` (correct) — the fix unifies both usages under the explicit dep field.

## Informational Observations

**[INFO-1] `normalizeSymbol` correctly NOT applied in `searchInstruments`**  
`searchInstruments` passes the raw `query` string to `this.client.search(query, ...)`. This is correct by design — Yahoo's `search()` takes a free-text query, not a symbol; passing `"BHP.AX"` instead of `"BHP"` would degrade autocomplete recall. The unit test at `yahooFinanceAuProvider.test.ts:288` explicitly documents this distinction: `"searchInstruments passes the bare query (NOT '.AX'-suffixed) to search()"`. Invariant 1 is satisfied; the in-code JSDoc calls this out. No action required.

**[INFO-2] `reserveCapacity(3)` flat pre-flight is conservative but documented**  
The pre-flight reserves 3 slots (bars + dividends + metadata) before any of the three fetch calls. This is a TOCTOU trade-off identical to the US/TW path as documented in the JSDoc referencing KZO-163 HIGH-1. The flat 3 causes over-reservation when metadata returns null (TW/US path), but that is intentional per the design. No action required.

**[INFO-3] Sydney TZ offset: always-on AEST (UTC+10) — AEDT caveat documented**  
`SYDNEY_TZ_OFFSET_MS = 10 * 60 * 60 * 1000`. Spike §4.2 caveat is carried in the JSDoc. The conservative AEST shift is correct for end-of-day bars (the AEST/AEDT 1-hour difference is well within the midnight-UTC safety margin for ASX close prices). No action required.

**[INFO-4] `fetchInstrumentCatalog` intentionally not rate-limited — JSDoc explains**  
`fetchInstrumentCatalog()` returns `[...AU_RESERVED_INSTRUMENTS]` synchronously without a remote call and without calling `assertCanConsume()`. This is intentional and documented in both the class JSDoc and the provider's rate-limit contract paragraph. No action required.

**[INFO-5] Wave 2 transition note correctly deferred**  
The scope-todo Phase 10 transition note (`docs/004-notes/kzo-172/transition-{datetime}-au-stock-ingestion.md`) is correctly deferred to Wave 2 / Technical Writer. The Phase 3 code review has no Wave 2 deliverables in scope. No action required.
