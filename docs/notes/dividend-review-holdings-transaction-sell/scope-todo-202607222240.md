# Todo: Dividend review, holdings selection, and SELL availability

> **For agents starting a fresh session:** read this entire file before implementation. This is the durable record of the locked scope; there was no debate note and there are no unresolved scope items.

## Locked Decisions

1. Replace the seven dividend-leg columns with two non-sortable stacked columns:
   - **Cash dividend:** currency-per-share rate, expected gross cash, received gross cash.
   - **Stock dividend:** normalized shares-per-share ratio and state, expected shares, received shares, share variance, and cash in lieu.
   - Keep provider-specific raw stock values and provenance in the review drawer.
   - Keep deductions, expected/actual net, cash variance, reconciliation status, and actions as separate columns.
2. Remove `expectedCashAmount`, `expectedGrossAmount`, and `receivedCashAmount` from review sorting in both the UI and API. The web route must normalize retired sort values to `paymentDate` and canonicalize the URL; direct API requests using retired sorts must be rejected.
3. Make review From/To controls match the transaction form's native date behavior: always editable, switch the preset to Custom when edited, and apply a complete valid range on blur.
4. Convert account, cash-status, and stock-status filters to checkbox multi-select dropdowns. Values are ORed within a filter and ANDed across filters; an empty selection means All. Wire URLs use repeated singular keys while internal DTOs use plural arrays. Deduplicate and cap each list at 50 values.
5. Fix Recent Receipts by sharing one desktop grid template between its header and rows without redesigning its columns or mobile layout.
6. Add a persisted holdings selection `none` mode shared by portfolio, dashboard, and report holdings tables. The picker footer always exposes Select all and Deselect all, disabling the already-active action. None mode shows `0 selected` and an intentional no-selection empty state.
7. Add eligible-share guidance to every shared new-transaction form: the global dialog, ticker dialog, and `/transactions` card. Existing transaction editing is out of scope.
8. Eligible shares means authoritative pre-trade inventory for the selected account, ticker, market, and trade date. Include existing same-day bookings before the new booking plus effective buys, sells, stock dividends, splits, and other position actions.
9. Add a dedicated sell-availability endpoint. Preview and transaction submission must call the same server-side as-of inventory calculation; submission reruns it against the freshest store immediately before persistence.
10. For SELL, show an inline Available to sell value, an explicit Use max action, and a native `max` constraint when known. Never auto-fill, silently clamp, or overwrite quantity.
11. Known oversells show an inline error and disable submission. Submission also rejects stale oversells with a stable insufficient-quantity error containing requested and available quantities.
12. Disable SELL submission while availability is loading. A transport failure shows a warning and permits a submission attempt. A successful authoritative `unavailable` result caused by invalid/unreplayable history blocks submission, and the write path fails closed for the same condition.
13. New and changed user-facing copy must be localized in English and Traditional Chinese.
